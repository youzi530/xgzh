## 一、产品定位回顾

| 维度 | 说明 |
|------|------|
| 产品名 | 新股智汇（XGZH） |
| 形态 | iOS / Android App + 微信小程序（一套代码 → 双端） |

## 四、用户旅程总览（典型场景）

### 场景 A：港股新股打新（核心场景）

```
用户登录 →（首页）看到"明日截止：XX 公司"卡片 → 点击进入新股详情
        → AI Agent 一键诊断（输出基本面+风险+情绪）
        → 切到"文章分析"看全网观点 TL;DR
        → 切到"历史相似股"看同行业首日表现
        → 决定申购 → 跳券商对比页
        → 选中富途/老虎，点击专属链接 → 唤起券商 App 开户/申购
        → 后台记录 CPA 转化
```

### 场景 B：CRS 报税新人

```
用户在首页看到"CRS 截止提醒" → 进入 CRS 向导
        → 完成 5 题身份判定 → 系统判定为"中国税务居民+香港金融账户"
        → 推送对应的 CRS 自我声明表填写指引
        → VIP 解锁完整模板包 → 下载 PDF
        → 返回个人中心设置每年 3 月提醒
```

### 场景 C：高频投资者（付费转化）

```
免费用户每日打开 → AI Agent 体验额度耗尽（5次/天）
        → 升级 VIP 弹窗（强调：无限 Agent + 历史数据全开）
        → 完成订阅 → 解锁高级功能
```

## 六、技术壁垒与防御点

| 壁垒类型 | 具体内容 | 防御策略 |
|----------|---------|---------|
| 数据壁垒 | 历史新股 10 年中签数据 + 文章语料库 | 持续爬取 + 数据清洗 SOP；版权风险用"摘要+原文跳转"规避 |
| AI 工程壁垒 | RAG 召回质量 + Tool Use 准确度 + Prompt 工程 | 离线评测集 + 在线 A/B + Bad case 闭环 |
| 合规壁垒 | "中立、不诱导、可追溯"的内容生成机制 | 全文档免责声明 + Output 关键词黑名单过滤 |
| 渠道壁垒 | 6+ 家主流券商 CPA/CPS 合作协议 | 业务团队 BD 优先，签独家或更高分成 |
| 用户体验壁垒 | "Agent + 文章 + 数据 + 券商" 一站式闭环 | 竞品多为单点，闭环本身就是护城河 |

## 七、关键非功能需求 (NFR)

| 维度 | 指标 | 备注 |
|------|------|------|
| 性能 | 首页 P95 < 1.5s；Agent 首字 < 1.2s | CDN+SSE 流式输出 |
| 可用性 | 月可用性 99.5% | 多可用区部署 |
| 成本 | 单次 Agent 对话成本 < ¥0.05 | 用 DeepSeek-V3 + Prompt 缓存 |
| 合规 | 全部页面带免责；用户数据加密；CRS/证券类内容仅做"信息汇总"不做"投资顾问" | 法律顾问 review |
| 可观测 | 全链路 Trace + 业务指标看板 | OpenTelemetry + Grafana |
