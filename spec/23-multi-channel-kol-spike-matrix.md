## 💡 关键洞见 (整合 6 个 sprint 经验 + 本次新增)

### 1. 用户提供的"AKShare 抓东财财富号"说法**不准确**, 但 AKShare 封装了**真正的金牛**

用户在 bug-fix-23:31 §方案二·2 提到 "AKShare 里有一个模块是专门抓取东方财富财富号和新浪财经创作者文章的". 实地 spike 发现:

- **AKShare `stock_news_em`** = ``so.eastmoney.com/news/s`` = **我已接入的 ``EastmoneySearchClient``**, 不是新东西
- **AKShare 没有**专门的 "财富号 / 创作者" 接口

但实地 spike 发现 AKShare 还封装了 3 个**未利用**的金融资讯接口:

| 接口 | 数据源 | 字段 | 限量 |
|---|---|---|---|
| `stock_telegraph` | 财联社电报 (cls.cn/telegraph) | 标题 / 链接 / 回帖数 | 实时 |
| `js_news(timestamp=)` | 金十数据 jin10.com | 资讯 / 时间 | 当日 4h 内 |
| `stock_zh_a_alerts_cls` | 财联社快讯 (cls.cn/searchPage) | 时间 / 快讯 | 单次前 10000 条 |

**财联社是金融 KOL 主战场** — 港股 + A 股 + 美股 IPO 几乎所有持牌分析师 + 自媒体大V 都在
财联社发声(连结 ``cls.cn`` 是同花顺旗下持牌资讯). pip install + 一行代码就能接入,
**ROI 远高于** 商业 API.

### 2. "钱解决问题" vs "工程解决问题" 的边界

Sprint 6.7 用户拍板"不付费"否决了新榜/清博/飞瓜, 但这次 bug-fix-23:31 §方案一 提到的
**WeChat Download API SaaS ¥19.9/月** 是另一个量级:

- 新榜 API: ¥1000+/月 (大 B 客户级)
- WeChat Download SaaS: **¥0.7/天** (个人开发者级)
- 阿里云 ShowAPI: ¥100/万次 (按量计费)

**¥0.7/天 vs 工程师维护反爬代码的边际成本**: 搜狗反爬每月触发 1-2 次, 每次至少 0.5 工程小时
处理(看日志 + 调节流参数 + 部署), 一年 12 工时 ≈ ¥800. WeChat SaaS 一年 ¥240, **省 ¥560**.

**Lesson** (待 Sprint 8 用户拍板): "不付费"原则**应区分量级** — 三位数/月 商业 API 必否决,
两位数/月 SaaS 应**按"省下的工程师工时"折算 ROI 评估**. 用户 Sprint 6.7 那次否决是"反对
¥1000 大 B 套餐", 不应推广到 ¥19.9 个人 SaaS.

### 3. 反爬强度等级表 (实地 curl 确认, Sprint 6.7-7.3)

| 强度 | 现象 | 案例 | 对策 |
|---|---|---|---|
| **L0** 无 | curl 直接 200 + 数据 | 智通 RSS / 新浪滚动 / 雪球 | 不需对策 |
| **L1** UA 检测 | curl 默认 UA 403, 假 Safari UA OK | 东方财富搜索 (Sprint 6.7) | 加 UA + Referer 即可 |
| **L2** IP 频率 | 连续 5+ 次 / 10s 触发 | 搜狗微信 (Sprint 6.9 / 7.3) | 节流 inter_query_delay 1.5→3.0s |
| **L3** 验证码 | 单次即重定向图形验证码 | **Baidu site:mp.weixin** (Sprint 7.3) | 不可行 |
| **L4** 登录态 | 无 cookie 直接拒 | 微信读书 / 微信"搜一搜" | 不可行 (无小号合规方案) |

**Lesson**: 任何代理/抓取候选源, 必须**单次 curl 实跳验证强度等级**. Sprint 7.3 把
spec/21 推荐的 Baidu fallback 5 分钟实跳就否决, 省 0.3d 假设接入工时.

### 4. 数据源**维度互补** > 数量堆砌

Sprint 8 后注册池将达 **9 源** (5 已接 + 1 长桥 + 4 新增), 但**类型分布**比"加更多源"重要:

| 维度 | 当前覆盖 | Sprint 8 后覆盖 |
|---|---|---|
| 持牌媒体 | 智通/新浪/东方财富 (3) | + 财联社/金十 (5) |
| 投资人社区 | 雪球 (1) | + 长桥社区 (2) |
| 微信公众号 | 搜狗 (1) | 同 |
| 持牌券商授权 | — | + 长桥 OpenAPI (1) |

每个维度至少 ≥ 2 源 → 单一源故障 (反爬触发 / 反代下线) 时 fallback 自然有候补.

### 5. Token-gated source 需要"灰度激活监控"

Sprint 7.3 已实施 `LongbridgeApiClient.is_enabled = bool(token)`, 但 Sprint 8 增多
token-gated 源后, 需要**单独的 metric** 暴露每个源的 enable/disable 状态:

```
GET /api/v1/admin/article_sources/status
→ [
    {"name": "智通 RSS", "enabled": true, "last_run_at": "...", "last_inserted": 5},
    {"name": "长桥 OpenAPI", "enabled": false, "reason": "LONGBRIDGE_API_TOKEN 未配置"},
    {"name": "WeChat Download SaaS", "enabled": false, "reason": "WECHAT_API_TOKEN 未配置"},
  ]
```

避免用户上线后**不知道哪些源没启用**, 误以为 ingest 异常. 这个 endpoint Sprint 8 可同时实施.
