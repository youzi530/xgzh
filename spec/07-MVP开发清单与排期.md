# 07 - MVP 第一阶段开发清单与优先级排期

> 目标：用 **10-12 周** 推出可对外发布的 MVP（小程序优先 + Android 内测 + iOS Beta）。
> MVP 验证假设：① "AI Agent + 新股诊断"是否 stick；② 券商导流是否能产生收入；③ 用户是否愿意为 VIP 付费。

---

## 一、MVP 范围界定（Scope Lock）

### 1.1 ✅ MVP 必做（P0）

| 模块 | 必做范围 |
|------|---------|
| 1. 新股 AI Agent | 单模型（DeepSeek-V3）+ 基础 Tool Use（5 个工具）+ 引用源 + 防幻觉护栏 |
| 2. 文章聚合 | 港股 + A 股 Tab；2-3 个数据源（雪球 + 智通 + 搜狗微信）；情感打标；TL;DR |
| 4. 券商对比 | 6-8 家主流券商；横向表格；专属邀请码导流；CPA 追踪 |
| 5. 历史新股数据 | 港 A 近 3 年数据；列表 + 散点图；AI 简易规律分析 |
| 6. 个人中心 + VIP | 注册登录 / VIP 月年订阅 / 邀请码 / 提醒设置 |
| 7. 首页 + 打新日历 | 今日/即将打新卡片 + AI 入口 + 打新日历 |
| L0 基础 | 鉴权、推送、埋点、监控、免责声明体系 |

### 1.2 🟡 MVP 后置（P1，第二阶段）

- CRS 报税向导（仅模板下载先做，完整向导和个税计算器排到第二阶段）
- 美股 IPO 模块
- 打新策略回测
- UGC 社区
- 多语言（仅简中先做）

### 1.3 ❌ MVP 不做

- iOS App 上架（先做 H5/小程序 + Android 内测，iOS 第二阶段过审）
- 复杂的会员等级体系（先做 VIP/非 VIP 二分）
- B 端 API 商业化

## 六、MVP 验收标准（DoD）

### 6.1 功能验收

- [ ] 用户可在小程序/H5/Android 完成注册登录
- [ ] 首页可见今日/即将打新卡片，打新日历正常滚动
- [ ] 新股详情页基础信息完整，AI 一键诊断 5 张卡片首屏 < 2s
- [ ] AI Agent 多轮对话流畅，引用源可点击查看
- [ ] AI Agent 命中黑名单关键词时正确兜底
- [ ] 文章列表港 A 双 Tab，至少 200 篇/天的覆盖
- [ ] TL;DR 一键生成，引用源可追溯
- [ ] 券商对比页 6 家以上，专属链接跳转可埋点
- [ ] 历史新股查询近 3 年覆盖完整
- [ ] AI 规律分析报告可生成
- [ ] VIP 月度/年度订阅可走通微信支付
- [ ] 邀请码绑定与奖励链路可走通

### 6.2 非功能验收

- [ ] P95 首屏 < 1.5s（H5/小程序）
- [ ] AI Agent P95 首字 < 1.2s
- [ ] AI 单次平均成本 < ¥0.05
- [ ] 后端服务可用性 ≥ 99%（监控验证）
- [ ] Sentry 错误率 < 1%
- [ ] 全部页面带免责声明，关键 CTA 上有合规提示
- [ ] 隐私政策、用户协议双 Check
- [ ] CPA 追踪报表可在后台查看 30 天数据

### 6.3 业务验收（上线后 4 周）

- [ ] 注册用户 ≥ 5,000
- [ ] 7 日留存 ≥ 20%
- [ ] AI Agent 日均调用 ≥ 1,000 次
- [ ] 券商点击 ≥ 500 次/天，至少 1 家有真实开户
- [ ] VIP 试用转化率 ≥ 8%
- [ ] 用户负反馈率 ≤ 5%

## 七、风险清单（红黄绿）

| 风险 | 等级 | 缓解措施 |
|------|:----:|---------|
| 模型价格上调 / 限流 | 🟡 | 多供应商接入，配置化切换 |
| 港交所 / 雪球 反爬 | 🟡 | 多源 fallback + 代理池 + 第三方 API 兜底 |
| 微信小程序审核（金融类目） | 🔴 | 提前与服务商沟通；不出现"建议买入"等敏感词；备齐主体资质 |
| 券商 BD 进度慢 | 🔴 | 第一批先选 2-3 家头部（富途/老虎/长桥），其他用公开邀请链接 |
| AI 输出违规导致投诉 | 🔴 | 关键词黑名单 + 输出后处理 + 用户协议免责 |
| iOS 审核拒（金融） | 🟡 | MVP 先不做 iOS，第二阶段做并准备齐资质 |
| RAG 效果不达预期 | 🟡 | 离线评测集 + Bad Case 闭环 + 顾问支持 |
| 团队带宽不足 | 🟡 | Scope 严格锁定，超出立刻砍 P1，优先保 P0 |
