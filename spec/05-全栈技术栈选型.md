> 目标：**一套代码同时跑通 iOS App / Android App / 微信小程序**；后端兼顾 AI/Agent 与高并发数据服务；数据库分层职责清晰；部署低成本可扩展。

## 一、前端框架选型：UniApp vs Taro

### 1.1 关键能力对比（2024-2025）

| 维度 | UniApp (Vue 系) | Taro (React 系) | 评估 |
|------|----------------|-----------------|------|
| **支持平台** | 小程序（微信/支付宝/抖音/百度...）+ App + H5 | 同 UniApp，覆盖一致 | 平 |
| **核心语言** | Vue 3 + TypeScript | React + TypeScript | 看团队偏好 |
| **App 编译方案** | UniApp 原生（HBuilder）/ uni-app x（强等同原生） | Taro RN / Taro Hybrid | UniApp 在国内 App 端更成熟 |
| **生态/插件市场** | DCloud 插件市场（小程序插件巨多，金融行业多） | npm 生态 + Taro 插件 | UniApp 国内生态略胜 |
| **小程序兼容度** | 高（DCloud 长期专注小程序） | 高（京东深度使用） | 平 |
| **性能** | uni-app x（uts）原生级 | Taro RN 原生级 | 平 |
| **TypeScript** | 良好 | 优秀 | Taro 略胜 |
| **状态管理** | Pinia（推荐）/ Vuex | Zustand / Redux Toolkit | 看偏好 |
| **UI 组件库** | uni-ui / uView 3.0 / Wot Design Uni | NutUI-React / Taroify | UniApp 现成金融模板更多 |
| **国内热度（2025）** | ★★★★★ | ★★★★ | UniApp 在国内中小团队更普及 |
| **学习成本** | 低（Vue 上手快） | 中（React + 小程序限制） | UniApp 略胜 |
| **官方维护活跃度** | 高（DCloud） | 高（京东） | 平 |
| **AI Coding 友好度** | 高（Vue 模板规则简单） | 高（JSX 灵活） | 平 |

### 1.2 推荐方案

> 🎯 **推荐：UniApp（Vue 3 + TypeScript + uni-app x）**
>
> 理由：
> 1. **小程序生态更成熟**，金融领域可直接复用（K线、表单、支付）；
> 2. **国内招聘市场 Vue 工程师供给充足**，团队扩张更容易；
> 3. **uni-app x（uts）** 编译为原生代码，性能接近 RN/Flutter，但学习成本低；
> 4. **DCloud 插件市场金融模板多**，缩短 MVP 周期。
>
> 如果团队已有重度 React 经验，则选 Taro 同样可行，差异不大。

### 1.3 前端技术栈细节

```
框架：       UniApp 4.x + uni-app x (uts)
语言：       TypeScript 5
状态管理：   Pinia
路由：       uni-app 内置 / uni-mini-router
UI 库：      Wot Design Uni（推荐，纯 Vue 3 + uni-app x 友好）
              + uView 3 兼用（部分组件更全）
图表：       uCharts（小程序优选）/ ECharts（H5/App）/ 自封装 Chart 组件
请求库：     uni.request 封装 + uni-axios（拦截器/重试）
富文本：     u-parse / mp-html
工具库：     dayjs、lodash-es、big.js（金融数值精度）
代码规范：   ESLint + Prettier + Stylelint + Husky + commitlint
构建：       Vite（H5）+ HBuilderX（App/小程序）+ uts 编译器
单测：       Vitest（业务逻辑）/ Playwright（H5 E2E）
监控：       Sentry SDK（小程序+App+H5）
埋点：       自建 SDK 或 GrowingIO/Sensors（视预算）
国际化：     vue-i18n + 自建 zh-CN/zh-HK/en 资源包
设计协作：   Figma → 蓝湖/即时设计 → 设计 Token 同步
```

### 1.4 关键工程实践

- **业务代码与平台代码隔离**：用 `condition compile`（`#ifdef MP-WEIXIN` / `#ifdef APP-PLUS`）只在必要处分支。
- **支付适配层**：抽 `services/payment.ts`，分别封装 Apple IAP / 微信支付 / 支付宝；上层只调 `pay(orderInfo)`。
- **路由统一**：封装 `navigateTo`，内部判断当前平台用合适的 API（小程序内嵌 H5 时降级）。
- **金额精度**：所有币种数值用 `big.js`，禁止 `parseFloat`。
- **暗黑模式**：用 CSS 变量 + `theme` Pinia store，全局动态切换。

## 二、后端框架选型

### 2.1 双语言混合架构（推荐）

```
┌─────────────────────────────────────────────────────────┐
│                BFF 网关 (Node.js / NestJS)               │
│   - 鉴权 / 限流 / 接口聚合 / 移动端友好                   │
│   - 业务无关、轻业务逻辑                                  │
└──────────────┬──────────────────────────────────────────┘
               │
       ┌───────┴────────┐
       ▼                ▼
┌────────────┐   ┌──────────────────────┐
│ 业务后端     │   │ AI Agent 服务         │
│ FastAPI     │   │ FastAPI + LangGraph  │
│ - IPO/CRS   │   │ - 对话/RAG/Tool Use   │
│ - 用户/订阅 │   │ - 流式 SSE 输出       │
│ - 券商/数据 │   │ - 离线评测/灰度       │
└────────────┘   └──────────────────────┘
```

### 2.2 选型理由

| 角色 | 选型 | 理由 |
|------|------|------|
| **BFF 网关** | Node.js 22 + NestJS 10 | 高并发、与前端 TS 同源、聚合接口/鉴权天然适合 |
| **业务后端** | Python 3.12 + FastAPI | 数据处理生态强（pandas/akshare/tushare 全在 Python） |
| **AI Agent** | Python + LangGraph + LiteLLM | LangGraph 编排 Agent 状态机；LiteLLM 统一多模型接入 |
| **爬虫** | Python + Scrapy / Playwright | 生态最全 |
| **任务队列** | Celery + Redis（或 RQ） | 处理爬虫、定时任务、长耗时 AI 任务 |
| **日程任务** | APScheduler / Celery Beat | 数据更新调度 |

### 2.3 BFF 网关核心职责

1. **鉴权与会话**：JWT 校验 + Redis Session（支持手机/微信/Apple）。
2. **接口聚合**：首页一次请求返回多模块数据，减少移动端往返。
3. **限流**：基于用户/IP/接口三维度滑动窗口限流（防爬/反刷）。
4. **协议转换**：把 Python 后端的内部 API 暴露为 RESTful + WebSocket（SSE）。
5. **客户端版本治理**：强制升级、灰度开关、兼容降级。

### 2.5 Agent 服务核心设计

```python
# 用 LangGraph 编排 Agent 状态机
from langgraph.graph import StateGraph

graph = StateGraph(AgentState)
graph.add_node('classify_intent', classify_intent)
graph.add_node('retrieve', hybrid_retrieve)
graph.add_node('llm_with_tools', llm_with_tools)
graph.add_node('execute_tools', execute_tools)
graph.add_node('post_process', validate_and_filter)

graph.add_edge('classify_intent', 'retrieve')
graph.add_edge('retrieve', 'llm_with_tools')
graph.add_conditional_edges('llm_with_tools', 
    lambda s: 'execute_tools' if s.tool_calls else 'post_process')
graph.add_edge('execute_tools', 'llm_with_tools')

agent_app = graph.compile()

# SSE 流式输出
@router.post('/agent/chat')
async def chat(req: ChatRequest):
    async def stream():
        async for chunk in agent_app.astream(req):
            yield f'data: {json.dumps(chunk)}\n\n'
    return StreamingResponse(stream(), media_type='text/event-stream')
```

## 三、数据库设计

### 3.1 数据库分层职责

| 库 | 职责 | 选型 | 备注 |
|----|------|------|------|
| **业务主库** | 用户/订阅/券商/订单 | PostgreSQL 16 | ACID、JSON 字段、生态成熟 |
| **缓存** | 热点数据/会话/限流 | Redis 7（Cluster） | 支持流式订阅 |
| **向量库** | RAG embedding | PGVector（MVP） → Milvus（量大后） | 与 PG 一体化省成本 |
| **全文检索** | BM25 / 文章搜索 | Elasticsearch 8 / Meilisearch | Meilisearch 更轻量 |
| **时序/分析** | 历史新股、K线、用户行为 | ClickHouse | 列存压缩、聚合极快 |
| **对象存储** | PDF/图片/导出文件 | 阿里 OSS / 腾讯 COS / S3 | CDN 加速 |
| **消息队列** | 异步任务/事件 | Redis Streams（轻量） / RabbitMQ / Kafka | MVP 用 Redis Streams |

## 五、安全与合规要点

| 维度 | 规范 |
|------|------|
| **传输加密** | 全站 HTTPS / WSS，TLS 1.3，HSTS |
| **存储加密** | 密码 bcrypt；手机号脱敏存储；密钥用 KMS |
| **接口签名** | 移动端关键接口加签名（防重放、防爬） |
| **JWT** | 短 Access Token（30min）+ 长 Refresh Token（30d）+ Redis 黑名单 |
| **数据脱敏** | 日志/导出禁止含完整手机号、身份证号 |
| **越权防护** | 所有资源访问校验 user_id 归属（行级权限） |
| **小程序合规** | 类目"金融-理财信息咨询"备案；隐私接口逐项申请 |
| **iOS 合规** | App Tracking Transparency；订阅必须用 Apple IAP |
| **Android 合规** | 个人信息保护合规白皮书；权限最小化 |
| **GDPR/PIPL** | 用户可导出/删除自己的数据接口 |
