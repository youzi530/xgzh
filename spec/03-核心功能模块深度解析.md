## 模块一、新股 AI Agent

### 1.1 用户操作流程

```
入口（首页/新股详情页/底Tab）
   │
   ▼
[1] 选择新股（搜索 / 列表点选 / OCR 招股书上传 [VIP]）
   │
   ▼
[2] AI Agent 自动诊断（首屏直出 5 张卡片）
       ├── 基本面卡（PE/PB/营收/净利）
       ├── 风险卡（红字 Top3 风险）
       ├── 多空情绪卡（看多 vs 看空 比例）
       ├── 上市预期卡（破发概率/首日涨幅区间）
       └── 同业对比卡（雷达图，3 家可比公司）
   │
   ▼
[3] 多轮追问（输入框 + 推荐问题 chips）
       例："对比腾讯历史首日表现"
           "PE 是否合理？"
           "保荐人过往业绩"
   │
   ▼
[4] 流式输出 + 引用源（每段下挂 [1][2] 来源链接）
   │
   ▼
[5] 操作（一键复制/分享/收藏/导出 PDF [VIP]）
```

### 1.2 UI 交互建议

| 元素 | 设计要点 |
|------|---------|
| **流式输出** | 仿 ChatGPT 打字机效果，首 token 1.2s 内必须吐出，避免空白等待 |
| **引用源** | 每段末尾 `[1] 招股书 P12`，点击弹层显示原文片段（防幻觉） |
| **推荐问题** | 顶部 4-6 个 chips，根据当前新股动态生成（如"为何 PE 高于行业"） |
| **免责声明** | 顶部固定半透明条："AI 输出仅供参考，不构成投资建议" |
| **Token 配额** | 免费用户 5 次/天，剩余用进度条可视化；耗尽时弹 VIP 升级页 |
| **错误兜底** | 模型限流/失败时，自动切换备用模型（GLM-4-Flash），并显示 toast |

## 模块二、新股文章分析与知识汇总

### 2.1 用户操作流程

```
[1] 进入"文章"Tab → 顶部分段控件 [全部 | 港股 | A股]
   │
   ▼
[2] 列表瀑布流（每条卡片）
       ┌────────────────────────────────┐
       │ [来源 Logo] 标题（最多2行）       │
       │ 摘要（最多3行）                  │
       │ 🟢看多 / ⚪中性 / 🔴看空  · 2h前  │
       │ #关联新股标签   阅读 · 分享      │
       └────────────────────────────────┘
   │
   ▼
[3] 顶部"AI 一键 TL;DR"按钮（针对当前筛选/某只新股）
       → 弹出底部 Drawer：
         · 多空比例饼图
         · 看多核心论据 Top3
         · 看空核心论据 Top3
         · 涉及来源列表（可逐条跳转）
   │
   ▼
[4] 点击文章 → 详情页（摘要 + 关键句高亮 + 跳原文按钮）
```

### 2.2 UI 交互建议

| 元素 | 设计要点 |
|------|---------|
| **情感标签** | 颜色化（绿/灰/红），右上角小圆点；详情页顶部大标签 |
| **TL;DR 入口** | 悬浮按钮，与底部 Tab 错开 24px 安全区 |
| **来源标识** | 来源 Logo 16×16，鼠标 hover/长按显示来源公信力等级 |
| **去重** | 同主题文章用"主文+N 篇相关"折叠展示 |
| **筛选** | 顶部支持按"新股 / 时间 / 情感 / 来源"多维筛选 |
| **首屏关怀** | 当无数据时提示"暂未抓取到 XX 公司公开文章，AI Agent 已为您启动深度分析" |

## 模块三、CRS 报税全流程指导

### 3.1 用户操作流程

```
[1] 入口（首页"工具"区 / 个人中心 / 全局搜索"CRS"）
   │
   ▼
[2] 身份判定问卷（5-7 步，每步带"为什么这么问"提示）
       Q1：你的国籍/护照？
       Q2：你过去 12 个月在哪里居住超过 183 天？
       Q3：你是否在 HK/SG/CN 持有金融账户？
       Q4：你是否在境外有商业活动/雇佣关系？
       Q5：你的金融账户余额是否曾超过 $25万 美元？
       …
   │
   ▼
[3] 系统输出"税务身份判定结果"
       例：「中国税务居民 + 香港金融账户持有者」
       同时给出后续步骤推荐路径
   │
   ▼
[4] 申报指南阅读（按地区 Tab：HK / SG / CN）
       · 时间节点（每年截止日）
       · 应申报内容
       · 例外情形
       · 常见问答 FAQ
   │
   ▼
[5] 模板下载中心
       · CRS 自我声明表（个人/实体）
       · W-8BEN（美股相关）
       · 各券商专属版本
       · 免费用户：缩略图预览；VIP：可下载 PDF/Word
   │
   ▼
[6] 个税辅助计算器
       · 输入：境外所得（股息/资本利得/利息）
       · 输出：估算应纳税额 + 抵免额（仅供参考）
   │
   ▼
[7] 设置每年提醒（自动添加到日历）
```

### 3.2 UI 交互建议

| 元素 | 设计要点 |
|------|---------|
| **问卷形式** | 全屏单题模式（一次只看一题），减少认知负荷；进度条 1/7 |
| **解释卡片** | 每题旁有 ⓘ 图标，点击显示"为什么这么问"+"涉及法规"摘要 |
| **判定结果** | 大色块卡片 + 推荐 3 个后续动作按钮（看指南/下模板/设提醒） |
| **风险提示** | 底部固定红色横条："本工具不构成税务/法律建议，最终请咨询专业人士" |
| **多地区切换** | 顶部 Pill 切换 HK/SG/CN，内容随之刷新 |
| **VIP 解锁动效** | 模板列表中，VIP 项目显示半透明遮罩 + 金色锁；点击触发升级页 |

## 模块四、券商对比（CPA/CPS 变现核心）

### 4.1 用户操作流程

```
[1] 进入"券商对比"
   │
   ▼
[2] 顶部分段：[港股券商 | A股券商 | 全部]
   │
   ▼
[3] 默认表格视图（横向滚动，固定首列券商名）
       维度（约 10 列）：
         · 佣金费率
         · 平台费 / 最低收费
         · 融资利率（保证金账户）
         · 撤单费 / 改单费
         · 港股打新中签率（近一年统计）
         · 新户活动（开户礼/入金礼）
         · 出入金便利度
         · 牌照（SFC / SEC / 中国证监）
         · 中文客服
         · 用户评分（社区聚合）
   │
   ▼
[4] 顶部按钮：[筛选] [排序]
       筛选：新户礼/打新支持/最低入金/暗盘交易
       排序：佣金升序、活动金额降序
   │
   ▼
[5] 点击某券商行 → 详情页
       · 完整费率明细 + 计算示例
       · 当前活动（带倒计时）
       · 用户真实评价（来源标注）
       · "立即开户"按钮（带专属邀请码 → UTM 追踪）
   │
   ▼
[6] 跳转
       App 端：调用 Universal Link 唤起券商 App
       小程序：跳转外部 H5（合规：标注"由 XX 提供"）
```

### 4.2 UI 交互建议

| 元素 | 设计要点 |
|------|---------|
| **表格** | 桌面式横滑表格，移动端友好；首列冻结，纵向滑动 |
| **关键数据高亮** | "最低佣金"、"最高入金礼"等用金色徽标突出 |
| **CTA 按钮** | "立即开户"按钮放在最显眼位置，使用品牌色；点击前埋点 |
| **活动倒计时** | 顶部红色倒计时条："富途新户礼剩余 3 天" |
| **真实感提升** | 显示"近 7 天有 X 位用户通过本站开户"（社会认同） |
| **合规提示** | 列表底部："券商信息由 XGZH 收集整理，可能有滞后，请以券商官网为准" |

### 7.3 全局原则

| 维度 | 规范 |
|------|------|
| **设计语言** | "金融专业 + 适度温度"；圆角 8/12，主色"信任蓝 + 强调金"，避免全亮黄/红 |
| **暗黑模式** | 必须支持；遵循 iOS/Android 系统切换；图表色板单独适配 |
| **字号无障碍** | 支持系统字号缩放；正文 14sp 起步 |
| **国际化** | 简中 / 繁中 / 英文（i18n 资源化，第一阶段先简中） |
| **加载态** | Skeleton 占位优于转圈；空数据态有插图 + 引导文案 |
| **错误兜底** | 网络/接口失败时给"重试"按钮，并显示友好文案；不暴露技术错误 |
| **埋点** | 所有 CTA 点击、模块曝光、AI 调用全埋点（含成本字段） |
