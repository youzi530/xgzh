## 风险 / 注意

- **DNS 传播**: 加 A 记录后 5min-2h, Caddy 第一次 Let's Encrypt 可能签失败几次, 这是正常的, 等 5min 重试就行
- **firewall 顺序**: 严格按 "先开 80/443, 测通, 再关 8000". 反了的话会把自己 SSH 锁出来 (虽然 22 一直开着, 但只是减少错误几率)
- **小程序 baseURL 改完**: 已经下载的小程序版本要重新 publish 一个版本才生效; 这一步是 P2-1 的一部分
- **DNS 改了, /etc/hosts 没改的本机会有缓存**: 必要时 `dscacheutil -flushcache` (Mac)
- **ops endpoint 是 P0 高敏感接口**: 必须强制走 X-Admin-Token (复用现有 `require_admin_token` 依赖); 不能让任何 JWT 通道接入
- **CORS_ORIGINS 改了要回写 GitHub Secrets**: 不能只改 ECS .env, 下次 deploy 会被 deploy.yml 覆盖
